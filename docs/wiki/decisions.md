# Decisions

## Phase 1 只做文本版 Chat Agent Demo

- 日期：2026-06-30
- 决策：第一阶段不实现真实录音、ASR、通知、小组件和 SQLite，先验证文本输入到结构化记录的闭环。
- 理由：当前可用模型接口是 OpenAI Compatible Chat Completions，尚未确认 ASR 服务；文本闭环能更快验证分类、时间识别和待确认机制。

## Phase 1 使用内存存储

- 日期：2026-06-30
- 决策：第一版记录保存在后端进程内存中。
- 影响：重启服务会丢失记录；适合 Demo，不适合长期试用。

## 数据库必须显式启用

- 日期：2026-07-02
- 决策：`vimo-go` 默认固定使用内存仓储；只有显式设置 `DB_DRIVER=mysql` 才连接 MySQL，单独存在 `MYSQL_DSN` 不触发数据库连接。
- 影响：本地 Demo 启动不再因为历史 `.env` 或环境变量里的 MySQL DSN 误连数据库；临时接回数据库时需要显式声明 `DB_DRIVER=mysql`。

## 意图理解由模型负责

- 日期：2026-07-01
- 决策：后端和前端不得通过本地关键词、短语表或正则判断用户意图；新增、查询、待确认续聊、确认授权、重复/相近和玩笑边界都由模型返回结构化 `intent`。
- 影响：代码层只做结构化校验、字段归一化、待确认结构合并、预览/自动保存等确定性执行逻辑；提示词负责描述全局语义规则。

## Agent 协议升级为意图栈

- 日期：2026-07-02
- 决策：Agent 输出从单意图扩展为 `primary_intent`、`secondary_intents`、`record_candidates`、`execution_plan` 和 `reply_strategy`，同时保留旧字段兼容现有前端和后端流程。
- 影响：主意图决定本轮主回复和主执行动作；副意图默认不自动执行，只展示为候选草稿、待确认或轻量提示。代码层只按结构化意图栈和风险字段做门控，不读取用户原文关键词。

## 待确认字段回答优先于同句副意图

- 日期：2026-07-02
- 决策：当 `open_contexts` 中的未收口项正在等待提醒时间、提醒开关或目标记录等高风险字段时，用户本轮给出的字段答案优先成为主意图 `update_pending`/`confirm_pending`；同句里的情绪、经历或日记价值作为副意图处理。
- 影响：避免把“八点吧，难受”这类补时间回答误判成“八点难受”的新日记；回复应先确认主任务，再轻量回应情绪或展示日记草稿。

## 借鉴 workflow state 模式但不做关键词路由

- 日期：2026-07-02
- 决策：Agent 协议新增 `intent_trace` 和 `pending_state/context_state`，借鉴 workflow state 的状态持久化、状态注入和确认门思想；业务代码仍不按用户原文关键词判断意图。
- 影响：每轮模型可以看到未收口项状态，输出结构化接续依据、风险原因、确认门原因和状态迁移，便于调试和 eval；前端只展示 trace chips，不把 trace 当成本地路由规则。

## 高风险动作使用结构化 Hard Stop Gate

- 日期：2026-07-02
- 决策：目标不唯一或低置信的修改/删除、模糊提醒时间、隐私日记、长期记忆和主动回访必须进入确认门；后端按模型 JSON 里的结构化字段执行 gate，不读取用户原文关键词。
- 影响：模型即使返回 `ready` 或 `auto_execute`，后端也会把命中的高风险项降为 `need_confirmation` 并写入 `intent_trace.gate_reasons`；前端会拦截目标不明确、目标低置信和其它 `hard_stop_*` 的静默执行。
- 2026-07-03 修订：已保存记录删除是可恢复软删除，AI 给出唯一 `target_id` 且目标置信足够时可以自动移入回收站；目标不唯一、目标缺失或低置信时仍进入删除确认卡。

## Agent 在线 Eval 已移除

- 日期：2026-07-02
- 2026-07-04 修订：暂时不需要独立 model-test / 在线 eval 流程，已删除 `vimo-go/cmd/eval`、`vimo-go/internal/agent/eval.go` 和 `vimo-go/internal/agent/evalcases/`。
- 影响：主服务、HTTP API、prompt 加载和前端主流程不再包含独立模型测试入口；`intent_trace` 仍保留为运行时结构化诊断字段，但不再有在线 eval runner 消费它。
